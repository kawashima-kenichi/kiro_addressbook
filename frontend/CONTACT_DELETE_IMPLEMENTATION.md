# 連絡先削除機能の実装

## 概要

タスク 9.5「連絡先削除機能の実装」を完了しました。この実装には、削除確認モーダル、30秒タイムアウト、成功・エラーメッセージ表示が含まれています。

## 実装されたコンポーネント

### 1. BaseModal.vue
再利用可能な確認モーダルコンポーネント

**機能:**
- カスタマイズ可能なタイトル、メッセージ、ボタンテキスト
- 30秒（デフォルト）の自動クローズタイムアウト
- カウントダウンタイマー表示
- 確認/キャンセルボタン
- 背景オーバーレイクリックでキャンセル
- Escapeキーでキャンセル
- Teleportを使用してbodyに直接レンダリング

**Props:**
- `isOpen: boolean` - モーダルの表示/非表示
- `title?: string` - モーダルのタイトル（デフォルト: "確認"）
- `message: string` - 確認メッセージ（必須）
- `confirmText?: string` - 確認ボタンのテキスト（デフォルト: "確認"）
- `cancelText?: string` - キャンセルボタンのテキスト（デフォルト: "キャンセル"）
- `autoCloseTimeout?: number` - 自動クローズまでの時間（ミリ秒、デフォルト: 30000）

**Events:**
- `confirm` - 確認ボタンがクリックされた時
- `cancel` - キャンセルボタンがクリックされた時、または背景がクリックされた時
- `close` - モーダルが閉じられた時

**テスト:**
- 11個のユニットテストが全て合格
- モーダルの表示/非表示
- カスタムプロパティ
- イベント発火
- タイムアウト機能
- タイマークリア

### 2. ContactCard.vue
連絡先カードコンポーネント

**機能:**
- 連絡先情報の表示（名前、電話番号、住所）
- 編集ボタン
- 削除ボタン
- 削除確認モーダルの統合
- カードクリックで詳細表示

**Props:**
- `contact: ContactDto` - 連絡先データ

**Events:**
- `edit` - 編集ボタンがクリックされた時
- `delete` - 削除が確認された時
- `view` - カードがクリックされた時

**削除確認メッセージ:**
```
[連絡先名]を削除してもよろしいですか？この操作は元に戻せません。
```

**テスト:**
- 7個のユニットテストが全て合格
- 連絡先情報の表示
- 電話番号なしの場合の処理
- 編集/削除/表示イベント
- 削除確認モーダルの動作

### 3. ContactListView.vue
連絡先一覧ビュー

**機能:**
- 連絡先一覧の表示
- 連絡先の削除（成功/エラーメッセージ付き）
- 連絡先の追加ボタン
- 連絡先数インジケーター
- 空状態メッセージ
- ローディング状態
- ログアウト機能

**成功メッセージ:**
```
連絡先[名前]が正常に削除されました
```

**エラーメッセージ:**
```
連絡先を削除できません。もう一度お試しください。
```

**メッセージ表示:**
- 5秒後に自動的に消える
- 緑色の背景（成功）
- 赤色の背景（エラー）

**テスト:**
- 9個のユニットテストが全て合格
- 連絡先一覧の表示
- 空状態の表示
- ローディング状態
- 削除成功/エラーメッセージ
- ナビゲーション

## 要件の検証

### 要件 5.1 ✓
削除ボタンをクリックすると確認ダイアログが表示される

### 要件 5.2 ✓
確認ダイアログで「確認」をクリックすると連絡先が削除される

### 要件 5.3 ✓
確認ダイアログで「キャンセル」をクリックすると削除がキャンセルされる

### 要件 5.4 ✓
削除成功時に確認メッセージ「連絡先[名前]が正常に削除されました」が表示される

### 要件 5.5 ✓
削除エラー時にエラーメッセージ「連絡先を削除できません。もう一度お試しください。」が表示される

### 要件 5.6 ✓
確認ダイアログは30秒間の非アクティブ状態後に自動的に閉じる

### 要件 5.7 ✓
削除を実行する前に明示的なユーザー確認を要求する
